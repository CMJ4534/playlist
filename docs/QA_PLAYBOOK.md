# MoodPlay 실사용 QA Playbook

> 목표: "실제 출시 가능한 안정성" — 사용자가 가장 먼저 겪을 문제를 중심으로 점검한다.

---

## 1. 핵심 이탈 포인트 (사용자 순서)

| 순서 | 흐름 | 이탈 가능성 | 점검 방법 |
|------|------|------------|-----------|
| ① | 앱 설치 → 온보딩 | 온보딩이 안 뜨거나 skip이 안 되면 즉시 삭제 | `onb-01`, `onb-02` |
| ② | 감정 선택 → 추천 요청 | 추천이 10초+ 걸리거나 에러 시 이탈 | `rec-01`, Edge timeout 확인 |
| ③ | 추천 결과 → 첫 곡 재생 | **오디오가 안 나오면 앱 삭제** (가장 치명적) | `play-01`, Actions > 3곡 큐 |
| ④ | 재생 중 다음 곡 전환 | 갭이 3초+ 이면 불안감 | `play-02`, skip rate 확인 |
| ⑤ | 앱 백그라운드 → 복귀 | 재생이 멈추고 안 돌아오면 이탈 | `bg-01`, `bg-02`, `bg-03` |
| ⑥ | 앱 종료 → 재시작 | 큐가 사라지면 재방문 의지 상실 | `pers-01`, `pers-02` |

## 2. 실기기 테스트 시나리오

### 시나리오 A: 첫 사용자 흐름 (Cold Start)
1. AsyncStorage 완전 초기화 (Beta QA > 도구 > 초기화)
2. 앱 재시작
3. 온보딩 3단계 → 완료
4. 홈 → 감정 선택 (예: 편안한)
5. 추천 로딩 → 결과 화면
6. **첫 곡 자동 재생 확인** (3초 내 오디오)
7. MiniPlayer 진행바 이동 확인
8. 다음 곡 버튼 → 전환 확인
9. 마지막 곡 → 플레이리스트 완료 표시

### 시나리오 B: 재방문 사용자
1. 앱 시작 (큐 존재 상태)
2. 홈 "이어듣기" 카드 표시 확인
3. 이어듣기 탭 → 기존 큐에서 재생 시작
4. 다른 감정 선택 → 새 추천
5. 이전 큐가 교체되고 새 큐 재생

### 시나리오 C: 에러 회복
1. Actions > embed 제한 테스트
2. 첫 곡 에러 → 자동 skip → 두 번째 곡 재생 확인
3. 10곡 연속 skip 테스트 → 에러 메시지 표시
4. Actions > queueRevision 강제 증가 → remount 정상 확인

### 시나리오 D: 백그라운드 복원
1. 재생 중 홈 버튼 (백그라운드)
2. 30초 대기
3. 앱 복귀 → 자동 resume 확인
4. 에러 상태에서 백그라운드 → 복귀 → resume 안 됨 확인

### 시나리오 E: 저장·공유
1. 추천 결과 → 저장 → toast 확인
2. 라이브러리 → 저장된 플레이리스트 표시
3. 상세 → 다시 듣기 → 재생
4. 감성 카드 공유 (PNG)

## 4. 플레이어 안정성 점검 기준

| 지표 | 목표 | 위험 | 점검 위치 |
|------|------|------|-----------|
| 첫 곡 재생 (3초 내) | 100% | <80% P0 | play-01 |
| 스킵률 | <20% | >35% P1 | Beta QA > 개요 |
| embed restricted | 0 | ≥5 P1 | 재생 탭 reason 분류 |

| timeout skip | <5% | — | reason breakdown |
| 완주율 | >30% | <15% P1 | 개요 |

## 5. 출시 판정 기준

### 🚫 BLOCK (출시 불가)
- P0 항목이 1개 이상
- 추천 성공률 <70%
- QA 체크리스트에 FAIL 존재
- 트랙 차단 비율 >30%

### ⚠️ WARN (조건부 출시)
- P1 항목 존재하지만 P0 없음
- 스킵률 >35%
- embed 제한 ≥5곡
- 완주율 <15%

### ✅ GO (출시 가능)
- P0/P1 없음
- 또는 P2만 존재

| Beta QA > 위험도 탭 | app/dev | P0/P1/P2 자동 평가 |
| Beta QA > QA 탭 | app/dev | 수동 체크리스트 |
| Player Snapshot | 도구 탭 | JSON 상태 덤프 |

## 7. 운영 모니터링 순서 (출시 후)

1. **D0**: 추천 성공률 ≥85%, 첫 곡 재생률 확인
2. **D1**: 스킵률, 완주율, fallback 비율 확인
3. **D3**: embed restricted 누적 확인 → 카탈로그 교체
4. **D7**: 재방문율, 저장/공유 사용률 확인
5. **매주**: 위험도 평가 실행 → BLOCK이면 hotfix

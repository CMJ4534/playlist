## 핵심 변경 포인트

### 1. QuickMoodRow 최상단 승격
- 현재: 홈 중간에 위치
- 제안: 홈 최상단 인사말 바로 아래 → **1탭 감정 선택**
- 이유: MoodPlay의 핵심 가치 = "감정 기반 큐레이션"

### 2. "지금 뜨는 플레이리스트" 섹션 신규
- 피드에서 engagement score 상위 2-3개 카드를 홈에 노출
- 탭하면 피드 상세로 이동
- "피드 더보기 →" CTA로 피드 탭 연결
- SNS 발견 UX의 진입점

### 3. HomeContinueCard 교체
- ~~현재 재생 중~~ → **"마지막 큐레이션"** 또는 제거
- 이유: 앱 내 재생이 없으므로 "현재 재생 중" 의미 없음
- 대안: "마지막으로 생성한 플레이리스트" 카드 (YouTube에서 열기 버튼 포함)

### 4. "YouTube에서 열기" 통합
모든 곡/플레이리스트에 "▶ 재생" 대신:
```
┌──────────────────────────────┐
│  ▶ YouTube에서 듣기           │  ← Primary action
│  🎵 Spotify에서 듣기          │  ← Secondary (Phase 2)
│  💾 내 라이브러리에 저장        │
│  📤 YouTube에 내보내기         │  ← Phase 2
└──────────────────────────────┘
```

### 5. 프로필 아이콘 추가
- 헤더 우측에 👤 프로필 아이콘
- Phase 1: 설정으로 연결
- Phase 3: 자신의 프로필 페이지로 연결

### 6. 팔로잉 활동 섹션 (Phase 3)
- 팔로잉한 사용자의 최근 활동
- 좋아요/댓글 알림 요약
- "더보기"로 알림 페이지 연결

## 탭 바 구조

### 현재
```
[ 홈 ]  [ 피드 ]
```

### 제안 (Phase 1)
```
[ 홈 ]  [ 피드 ]  [ 라이브러리 ]
```

### 제안 (Phase 3)
```
[ 홈 ]  [ 탐색 ]  [ + 만들기 ]  [ 라이브러리 ]  [ 프로필 ]
```

- **홈**: 개인화된 추천 + 활동 피드
- **탐색**: 감정별 인기 플레이리스트 + 크리에이터 발견
- **+ 만들기**: 감정 선택 → 플레이리스트 생성 (기존 emotion flow)
- **라이브러리**: 저장/좋아요/내가 만든 플레이리스트
- **프로필**: 내 활동, 팔로워, 설정

## UX 원칙

1. **Emotion-first**: 모든 진입점은 감정에서 시작
2. **Discovery over playback**: 발견·저장·공유가 핵심 루프
3. **외부 재생 위임**: 앱 내 재생 UI 최소화
4. **Social proof**: 좋아요/저장 수로 플레이리스트 신뢰도
5. **개인화**: 감정 히스토리 기반 홈 커스터마이징
6. **다크 감성 유지**: 현재 moodTheme 100% 활용
